The user wants to build a fully autonomous AI-to-AI Venture Capital (VC) fund platform, Anima Fund. This involves a multi-agent platform to create, monitor, and manage multiple independent AI agents from a single UI. Each agent must be fully autonomous, operating in its own sandboxed VM on the live internet.

**PRODUCT REQUIREMENTS:**
-   **Core Functionality:** A multi-agent platform to create, monitor, and manage independent AI agents from a single dashboard. Each agent has its own VM, wallet, and skills.
-   **Agent Autonomy & Lifecycle:**
    1.  The agent must be created and operate *exclusively within* a sandboxed VM with no access to the host.
    2.  The agent's operations are governed by a strict, mechanically-enforced 4-phase prompt, starting with capital acquisition.
-   **Real-time Dashboard:** A dashboard to monitor all agents, with real-time logs and terminal/browser views for each Anima VM.
-   **User Control:** The user provisions the agent's tools and seed funding but does not control the agent's actions once it goes autonomous.

**User's preferred language**: English

**what currently exists?**
The application consists of a dashboard for managing existing agents and a standalone Genesis screen for provisioning new ones. The architecture has been significantly refactored to be per-agent, meaning each agent has its own Conway API key stored in its MongoDB record, ensuring data isolation for all API calls and monitoring.

The provisioning flow is a multi-step wizard on the Genesis screen where a user can:
1.  Connect their Conway account by providing an API key.
2.  Select from a list of VM tiers.
3.  Execute a 6-step process to create a sandbox VM and deploy the agent into it.

The UI supports navigating between the dashboard and genesis screen, creating new agents via a modal, and selecting which agent to manage or provision. The agent deployment process now includes a Phase 0 self-test where the agent verifies its own tools (browser, OpenClaw, etc.) upon startup.
