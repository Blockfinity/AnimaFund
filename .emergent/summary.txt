**Key Technical Concepts**
-   **Full Agent Isolation:** All backend endpoints are now fully context-aware, reading all data (state DB, logs, wallet, ) from the currently selected agent's directory (). This is the cornerstone of the multi-agent system.
-   **Dynamic Skill Aggregation:** The backend's  endpoint dynamically combines skills from local files (Anima skills), introspection of the Conway engine (native tools), and simulated web calls (ClawHub marketplace) into a single, unified list for the frontend.
-   **State Persistence Across Restarts:** The system now uses a file () to store the name of the currently active agent. This ensures that server hot-reloads do not reset the application's context to the default agent.
-   **Live Agent Updates:** The  endpoint allows for pushing configuration file updates (like the constitution) to the directories of running agents without requiring a full reset or restart.
