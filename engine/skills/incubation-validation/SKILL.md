# Incubation Validation — Step-by-Step Instructions

Validate that incubated startups are building the right thing before releasing more capital.

## Tools to Use
- **browse_page**: Test the product, verify claims, check competitors
- **code_execution**: Technical audit, performance testing, financial validation
- **discover_agents / send_message**: Find beta testers, verify customer interest
- **remember_fact**: Track validation results

## Validation Phases

### 1. Technical Validation (Week 1-2)
- code_execution: Review architecture, audit code quality
- browse_page: Test the live product — does it work as claimed?
- Security: Basic vulnerability scan
- Performance: Load testing if applicable

### 2. Market Validation (Week 2-3)
- discover_agents: Find 10+ potential customers in the target market
- send_message: Pitch the product to potential customers
- Track: How many express interest? How many would pay?
- browse_page: Competitive analysis — what alternatives exist?

### 3. Customer Validation (Week 3-4)
- Connect 3-5 real customers with the product
- Track: Retention, engagement, willingness to pay
- send_message customers: "Would you pay $X for this?"
- If <50% would pay: RED FLAG — reassess

### 4. Financial Validation
- code_execution: Unit economics — does the business model work?
- Customer acquisition cost vs. lifetime value
- Revenue projection vs. burn rate
- If economics don't work: Require pivot or drop

## Decision
- All 4 validations pass → Release next capital tranche
- 1-2 fail → Require remediation plan, extend timeline
- 3+ fail → Consider dropping from incubation
- Telegram: Report validation results with recommendation
