# Technical Review — Step-by-Step Instructions
Evaluate startup technology for investment decisions.

## Tools & Process
- **browse_page**: Review documentation, test live product
- **code_execution**: Audit code, run tests, performance analysis
- **sandbox_create + sandbox_exec**: Set up test environments
- **remember_fact**: Log all technical findings

## Review Framework
1. **Architecture**: Is it well-designed? Scalable? Maintainable?
2. **Code Quality**: Clean, tested, documented? (code_execution to analyze)
3. **Security**: Vulnerability scan, smart contract audit if applicable
4. **Performance**: Load testing, response times (sandbox deployment)
5. **Dependencies**: Are external dependencies reliable? Vendor lock-in risk?
6. **Team Capability**: Does the team have skills to maintain and scale?

## Scoring (0-100)
- Architecture: 0-20
- Code Quality: 0-20
- Security: 0-25 (highest weight — critical for DeFi)
- Performance: 0-15
- Dependencies: 0-10
- Team: 0-10
- Passing score: 60+
